## Today's Changes (5 optimizations/fixes)

### 1. ASR WS partial/final/reset type field (3c04247)

Added explicit `"type"` field to WebSocket ASR stream messages so clients can distinguish partial from final from reset. Previously only the final message was distinguishable by empty payload.

### 2. ASR final emission fix (3cb6f3f)

Streaming ASR final text now runs full offline decode on the accumulated audio instead of using the last partial result. Ensures final text quality matches offline `/asr` endpoint.

### 3. ASR partial dedup + VAD endpoint 1000ms (0544525)

Token-level boundary dedup (`_dedup_boundary_tokens`) to remove partial→partial overlap. VAD endpoint silence threshold relaxed from 500ms → 1000ms to reduce false endpointing on pauses.

### 4. P0 #906: ORT CUDA stream isolation (938d240)

**Root cause**: ONNX Runtime's CUDA EP was using the default legacy CUDA stream, which conflicted with TRT's CUDA graph capture. TRT capture claims the legacy stream, and any ORT inference on that stream during capture causes undefined behavior (crashes, graph corruption).

**Fix**: Create a dedicated non-blocking CUDA stream for ORT's `user_compute_stream`, isolating ORT from TRT's stream entirely.

**Verified**: 5 consecutive V2V runs with no crash (previously crashed on run 2).

### 5. CHUNK_SIZE 250ms → 400ms (be0ef76)

**Why**: Larger chunk size reduces encoder invocations per utterance (~11 → ~7 for 2.8s audio), which:
- Reduces partial emission frequency (3× fewer partials)
- Improves latency (fewer encode/decode cycles)
- Maintains final text accuracy

**Verified**: 3 wavs (S0/S1/S3) — partial quality improved, final text unchanged, 5-run V2V sweep all correct.

## Caveats & Known Issues

1. **Image rebuild** — If rebuilding the Docker image from scratch, remember to:
   - `pip install cuda-python` (needed for `_get_asr_cuda_stream_handle`)
   - Remove `transformers` from the image (it's not needed with the C++ TRT pipeline)
2. **Partial display still not perfect** — First partial on short utterances still can hallucinate ("当前。") before sufficient audio context. CHUNK_SIZE increase helps but doesn't eliminate this. Potential future work: suppress partials until enough audio accumulated (>1s).
3. **V2V warm variance** — Runs 2-3 after container restart can show higher latency (up to 2×) before GPU thermal state stabilizes. Not a bug, just thermal characteristics of Orin NX passive cooling.
4. **orin-nx vs desktop comparison not done** — No baseline from recomputer-desktop for this V2V pipeline. Future task.
5. **No TTS backend changes today** — TTS T1 cache commit not in this branch. All work focused on ASR pipeline.
